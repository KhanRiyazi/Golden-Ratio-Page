## 💰 Why This Project Earns Money

Most freelance / agency sites fail because they look generic and talk about themselves. DataMind is built around three conversion principles used by top-earning consultants:

### 1 · Show Evidence, Not Claims
Every service has matching case studies with specific, verifiable metrics (31% churn reduction, $2.4M ARR saved). Clients don't buy capability — they buy proof.

### 2 · Reduce Risk Before the Call
The 5-step process section answers "what happens if I hire them?" before the client even asks. Showing a discovery phase + proof of concept gate removes the biggest objection: fear of wasted money.

### 3 · Interactive Demo Creates Visceral Value
The live churn-risk model widget lets prospects *feel* what your work does before any conversation happens. It is the single highest-converting element on the page — it demonstrates expertise in 30 seconds.

## 📧 EmailJS Setup — Connect Your Inbox in 5 Minutes

EmailJS lets the contact form deliver real emails to your inbox with **zero backend code**.

### Step 1 — Create Your Account
Go to [emailjs.com](https://www.emailjs.com) → Sign up free (200 emails/month on free tier).

### Step 2 — Add an Email Service
```
Dashboard → Email Services → Add New Service
Select Gmail / Outlook / SendGrid / etc.
Authenticate your email account
Copy the Service ID  →  e.g. service_abc123
```

### Step 3 — Create an Email Template
```
Dashboard → Email Templates → Create New Template
```

Paste this template body (copy exactly):

```
New project enquiry from DataMind website

Name:     {{from_name}}
Email:    {{reply_to}}
Company:  {{company}}
Service:  {{service_type}}

Message:
{{message}}

---
Reply directly to this email to reach the sender.
```

Set **Reply To** field to: `{{reply_to}}`

Copy the Template ID → e.g. `template_xyz789`

### Step 4 — Get Your Public Key
```
Dashboard → Account → API Keys → Public Key
Copy it  →  e.g. AbCdEfGhIjKlMnOp
```

### Step 5 — Paste Into script.js

Open `script.js` and find lines ~364–366:

```javascript
// BEFORE (placeholders)
const EMAILJS_PUBLIC_KEY  = 'YOUR_PUBLIC_KEY';
const EMAILJS_SERVICE_ID  = 'YOUR_SERVICE_ID';
const EMAILJS_TEMPLATE_ID = 'YOUR_TEMPLATE_ID';

// AFTER (your real keys)
const EMAILJS_PUBLIC_KEY  = 'AbCdEfGhIjKlMnOp';
const EMAILJS_SERVICE_ID  = 'service_abc123';
const EMAILJS_TEMPLATE_ID = 'template_xyz789';
```

### Step 6 — Test It
Fill the contact form and hit Send. Check your inbox within 30 seconds.

### Template Variable Map

| HTML `name` attribute | EmailJS template variable | What it contains |
|---|---|---|
| `from_name` | `{{from_name}}` | Sender's full name |
| `reply_to` | `{{reply_to}}` | Sender's email (also Reply-To) |
| `company` | `{{company}}` | Company name (optional) |
| `service_type` | `{{service_type}}` | Selected dropdown value |
| `message` | `{{message}}` | Project description |

## 🎨 Customisation Guide

### Change Brand Colours

All colours are CSS custom properties in `style.css` lines 1–20:

```css
:root {
  --gold:       #c8a96e;   /* ← primary accent — change to your brand */
  --gold-light: #e0c98a;   /* hover state of gold */
  --teal:       #2dd4bf;   /* secondary accent (step durations, etc.) */
  --bg:         #080c12;   /* page background */
  --ink:        #f0f4ff;   /* primary text */
}
```

**Common brand swaps:**

| Style | `--gold` value | `--teal` value |
|---|---|---|
| Electric Blue | `#4f8ef7` | `#7dd3fc` |
| Emerald | `#34d399` | `#6ee7b7` |
| Rose / Pink | `#f472b6` | `#e879f9` |
| Corporate Navy | `#3b82f6` | `#60a5fa` |

### Change Company Name & Copy

1. **Brand name** — search `DataMind` in `index.html` (appears in nav, footer, `<title>`)
2. **Tagline** — update `.hero-sub` paragraph text
3. **Email address** — update `hello@datamind.io` in the contact section + footer
4. **Stats** — update `data-target` values on `.stat-num` elements:

```html
<span class="stat-num" data-target="94">0</span>   <!-- 94% -->
<span class="stat-num" data-target="3">0</span>    <!-- 3x  -->
<span class="stat-num" data-target="47">0</span>   <!-- 47+ -->
```

### Change Typed Phrases

In `script.js` section 3:

```javascript
const phrases = [
  'competitive edge.',      // ← edit any of these
  'product decisions.',
  'revenue uplift.',
  'customer insight.',
  'operational clarity.',
];
```

### Swap Case Studies

Each case study is an `<article class="case-card">` in `index.html`. Edit:
- `data-category` — `ml` | `nlp` | `analytics` | `engineering`
- The `<h3>` title
- The `<p>` description
- `.metric-val` values
- `.case-stack` tech badges

### Add / Remove Services

Services are `.service-card` divs inside `.services-grid`. The grid auto-adjusts for 2, 4, or 6 cards. Each card follows this pattern:

```html
<div class="service-card reveal">
  <div class="service-icon">
    <svg><!-- your icon --></svg>
  </div>
  <h3>Service Name</h3>
  <p>One-paragraph description.</p>
  <ul class="service-bullets">
    <li>Bullet one</li>
    <li>Bullet two</li>
  </ul>
</div>
```

## 🐛 Troubleshooting

### Form Sends But No Email Arrives

```
1. Check spam / promotions folder
2. Verify template variables match exactly:
   HTML name=     →   Template {{var}}
   from_name      →   {{from_name}}
   reply_to       →   {{reply_to}}
   company        →   {{company}}
   service_type   →   {{service_type}}
   message        →   {{message}}
3. Check EmailJS dashboard → Email History for send attempts
4. Ensure your domain is in EmailJS Allowed Origins
```

### Error: 401 Unauthorised

```
Your EMAILJS_PUBLIC_KEY is wrong.
Dashboard → Account → API Keys → copy Public Key (not Secret)
```

### Error: 403 Forbidden

```
Your domain is not in the EmailJS allowlist.
Dashboard → Account → Security → Allowed Origins → add your URL
(Include both http:// and https:// variants during development)
```

### Error: 404 Not Found

```
EMAILJS_SERVICE_ID or EMAILJS_TEMPLATE_ID is wrong.
Double-check both IDs — they are case-sensitive.
```

### Error: 422 Unprocessable

```
Template variable mismatch.
Your EmailJS template uses {{from_name}} but the JS sends from_name — they must match exactly.
Check for typos, extra spaces, or capitalisation differences.
```

### Particles Not Showing

```
Canvas requires offsetWidth > 0 at init time.
Make sure the hero section has height before JS runs.
The section has min-height: 100vh — this should be automatic.
If using display:none to hide sections, particles won't render.
```

### Fonts Look Wrong

```
Check your internet connection — fonts are loaded from Google Fonts.
For offline use, download fonts from fonts.google.com and host locally.
Update the @import in <head> to local paths.
```

### Mobile Menu Not Closing

```
All mobile links must have class="mob-link".
Check that your custom links include this class.
```
